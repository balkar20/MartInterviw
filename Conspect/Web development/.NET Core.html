.NET Core


Термины
  • IDE — интегрированная среда разработки (MS Visual Studio и др.)

  • .NET Framework — платформа, включающая CLR (общеязыковую исполняющую среду), BCL (Base Class Library), FCL (Framework Class Library). .NET Framework = CLR + BCL + FCL. Архитектура платформы описывается спецификацией CLI (Common Language Infrastucture). .NET Framework изначально проектировалась как кросс-платформенная, но затем Microsoft изменила приоритеты и на данный момент эта платформа работает только в среде Windows.

  • .NET Core — Цель: отделение .NET Framework от ОС Windows. Microsoft удалили из .NET Framework те компоненты, которые не являлись ядром и переработали CLR. В результате появилась кросс-платформенная общеязыковая среда CoreCLR и набор библиотек классов CoreFX (функциональность которых частично совпадает с BCL). Платформа .NET Core стала существенно меньше, чем .NET Framework, потому что многие компоненты были удалены, например, Windows Forms и WPF, потому что они были тесно связаны с ОС Windows, ASP.NET Web Forms, WCF. ASP.NET MVC и ASP.NET Web API были переработаны и включены в .NET Core. В .NET Core также входит кросс-платформенная технология доступа к реляционным данным Entity Framework Core..NET Core распределеляет организует компоненты в виде NuGet-пакетов, которые могут быть развернуты независимо друг от друга (примеры NuGet-компонентов: CoreCLR, JIT-компилятор, System.Console и тд.). Задача: разделить .NET Core на компоненты и уменьшить количество зависимостей, чтобы для развертывания требовались только те компоненты, которые необходимы для работы приложения. Кросс-платформенность обеспечивается за счет механизма двухэтапной компиляции: промежуточный код может компилироваться кросс-платформенной исполняющей средой CoreCLR в нативный код требуемой ОС.
    Особенности: 
    - кросс-платформенность (Windows, Mac, Linux) — разработка и развертывание может быть выполнена в разных ОС;
    - открытый исходный код (Open source);
    - реализует .NET Standart;
    - совместимость с .NET Framework, Xamarin и Mono через библиотеку .NET Standart;
    - гибкое развертывание (FDD, SCD);
    - модульность;
    - инструменты командной строки (.NET Core CLI);

  • .NET Standart (стандартизированный набор API) — спецификация набора API для BCL, который должен быть доступен во всех .NET платформах (.NET Core, .NET Framework, Xamarin), которая призвана обеспечить совместимость между разными платформами, что позволит упростить процесс совместной работы над кодом на разных платформах .NET; позволяет создавать переносимые библиотеки, которые могут использоваться в разных реализациях .NET. Другими словами, эта спецификация унифицирует базовые библиотеки классов (BCL) разных платформ .NET, посредством создания единого API, реализуемого всеми платформами .NET. (.NET Standart — это интерфейс, API, .NET Framework, .NET Core, .Mono — реализации стандарта).

  • .NET Native

  • Mono — .NET Core отличается от Mono тем, что не имеет реализаций некоторых компонентов (например, Windows Forms) и не реализует API .NET Framework с использованием тех же имен сборок.

CLR (общеязыковая исполняющая среда) — исполняющая среда для байт-кода (MSIL - промежуточный язык, высокоуровневый ассемблер). Компилятор переводит программы, написанные на .NET-совместимых языках (C# - компилятором C#, C++ - компилятором C++, VB .NET компилятором VB.NET) в MSIL (байт-код), получая сборки (.dll — библиотека, .exe — исполняемая сборка). Затем байт-код исполняется средой CLR с помощью встроенного в нее JIT-компилятора (динамическая компиляции во время выполнения, единожды скомпилированный таким образом код кэшируется), байт-код преобразуется в машинный код (нативный) для конкретного целевого процессора. Среда CLR включает в себя виртуальную машину CLR. Функции CLR: загрузка пользовательских сборок, управление памятью, сборка мусора, управление кодом, безопасность кода, управление работой потоков и другое.

FCL (Framework Class Library) — библиотека классов платформы. Предоставляет классы  Windows Forms, ADO.NET, ASP.NET, LINQ, WPF, WCF и другие. 

BCL (Base Class Library) — стандартная библиотека классов платформы, является ядром FCL, то есть BCL ⊂ FCL. В основном в нее входят классы из пространства имен System.

Существуют 2 типа развертывания приложений: 
  1. Framework-dependent deployment — FDD — Развертывание, зависящее от инфраструктуры:
    на целевом компьютере должна быть установлена определенная версия .NET Core и при запуске вашего приложения будут загружены CoreCLR и CoreFX из папки фреймворка (ваше приложение зависит от общесистемной версии .NET Core);
  2. Self-contained deployment — SCD — Автономное развертывание :
    абсолютно все зависимости(.NET Core-платформа (CoreCLR, CoreFX) и сторонние библиотеки) поставляются в папке с самим приложением.

Модульность в .NET Core — .NET Core доступен в виде NuGet пакетов. Модульный подход позволяет разработчикам оптимизировать приложение включая только те зависимости, которые необходимы для работы приложения. CoreFX теперь содержит гораздо больше сборок, чем было в BCL .NET Framework, но они стали существенно меньше. Скажем, если в .NET Framework требовался класс из пространства имен System.IO, то к приложению подгружалась тяжеловесная сборка System.dll, которая содержала 10-ки других пространств имен и весила 1,5 Мб. В .NET Core, в аналогичной ситуции будет подгружена сборка SystemIO.dll, размером всего 22Кб. Из-за модульности приложения на платформе .NET Core будут весить меньше, так как не включают в себя ненужные зависимости. 

В .NET Core можно создать проекты следующих типов: Консольное приложение, Веб-приложение ASP.NET Core, Библиотека классов (.NET Core). Также можно создать Библиотеку классов .NET Standart, которую можно использовать на любой платформе .NET, реализующей спецификацию .NET Standart.

.NET Core CLI — Command Line Interface (Интерфейс командной строки) — используя .Net Core CLI можно создавать разные типы проектов, компилировать, запускать, добавлять ссылки на зависимости и прочее прямо из консоли.

dotnet new console -output myConsoleApp
Создать консольное приложение Hello World в папк е myConsoleApp
cd myConsoleApp

dotnet run
Запустить проект
dotnet build
Собрать проект
dotnet new library -output myLibrary
Создать библиотеку .Net Core
dotnet add reference ../myLibrary/myLibrary.csproj
Добавить ссылку в myConsoleApp на библиотеку myLibrary
